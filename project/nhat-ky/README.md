# 📓 Nhật Ký Phát Triển — Nodi POS & Platform

> Mỗi file = 1 ngày làm việc. Entry mới nhất ở đầu file.

## Quy tắc

1. **Tên file**: `YYYY-MM-DD.md` (VD: `2026-03-18.md`)
2. **Mỗi phiên** = 1 section `## HH:MM — Tiêu đề`
3. **Ghi TẤT CẢ**: thành công, thất bại, bugs, bài học, quyết định
4. **Agent mới**: đọc file ngày hôm nay + 2 ngày gần nhất để nắm context

## Template

```markdown
# 📓 Nhật ký — DD/MM/YYYY

---

## 🔧 HH:MM — Tiêu đề phiên làm việc

### Mục tiêu
### Kết quả
### Files thay đổi
### Build
### Bài học (nếu có)
```
