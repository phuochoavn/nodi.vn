## 📦 Phase 1: Nhận Dữ Liệu Sync (Priority High)

### Database Schema (Supabase/PostgreSQL)

```sql
-- Master: Thông tin cửa hàng
CREATE TABLE stores (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    license_key TEXT UNIQUE,
    hwid TEXT,
    store_name TEXT,
    store_address TEXT,
    store_phone TEXT,
    last_sync_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Sync Data Tables (prefix: sync_)
CREATE TABLE sync_customers (
    id BIGINT,
    store_id UUID REFERENCES stores(id),
    name TEXT, phone TEXT, address TEXT,
    current_debt DECIMAL, credit_limit DECIMAL,
    cccd TEXT, created_at TIMESTAMPTZ,
    PRIMARY KEY (store_id, id)
);

CREATE TABLE sync_products (
    id BIGINT,
    store_id UUID REFERENCES stores(id),
    name TEXT, barcode TEXT, category TEXT,
    base_unit TEXT, stock_quantity DECIMAL,
    cost_price DECIMAL, sell_price DECIMAL,
    expiry_date DATE,
    PRIMARY KEY (store_id, id)
);

CREATE TABLE sync_product_units (
    id BIGINT,
    store_id UUID REFERENCES stores(id),
    product_id BIGINT,
    unit_name TEXT, exchange_value DECIMAL, price DECIMAL,
    PRIMARY KEY (store_id, id)
);

CREATE TABLE sync_orders (
    id BIGINT,
    store_id UUID REFERENCES stores(id),
    invoice_number TEXT,
    customer_name TEXT, customer_phone TEXT,
    total_amount DECIMAL, discount_amount DECIMAL, final_amount DECIMAL,
    payment_method TEXT, status TEXT,
    created_at TIMESTAMPTZ,
    PRIMARY KEY (store_id, id)
);

/* ... Similar tables for:
   - sync_order_items
   - sync_suppliers
   - sync_purchase_orders
   - sync_purchase_items
   - sync_customer_transactions
   - sync_supplier_transactions
   - sync_store_funds
   - sync_cash_transactions
   - sync_product_batches
   - sync_payment_vouchers
   - sync_store_settings
*/
```

### API Endpoints

```
POST /api/sync          - Nhận full payload từ app
POST /api/backup/create - Tạo bản backup mới
GET  /api/backup/list   - Danh sách backups
GET  /api/backup/{id}   - Download backup để restore
```

## 💾 Phase 2: Backup & Restore (Priority Critical)

### Backup Strategy

| Loại | Tần suất | Retention |
|------|----------|-----------|
| **Full Backup** | Mỗi lần sync | 30 ngày |
| **Incremental** | (Future) | N/A |

### Restore API

```typescript
// Request: GET /api/backup/{backup_id}
// Response: Full SyncPayload JSON

interface RestoreResponse {
  backup_id: string;
  created_at: string;
  data: SyncPayload;  // Same format as sync upload
}
```

### Storage Strategy

- **Hot Storage**: Supabase DB (7 ngày gần nhất)
- **Cold Storage**: Supabase Storage/S3 (JSON compressed, 30 ngày)

## 📊 Phase 3: Analytics Dashboard (Priority High)

### Business Intelligence Metrics

#### Per Store Analytics

```sql
-- 1. Doanh thu theo ngày/tuần/tháng
SELECT store_id, DATE(created_at), SUM(final_amount) as revenue
FROM sync_orders GROUP BY 1, 2;

-- 2. Top sản phẩm bán chạy
SELECT store_id, product_name, SUM(quantity) as sold
FROM sync_order_items GROUP BY 1, 2 ORDER BY 3 DESC LIMIT 10;

-- 3. Tình trạng công nợ
SELECT store_id, SUM(current_debt) as total_debt
FROM sync_customers GROUP BY 1;

-- 4. Tồn kho cần nhập
SELECT store_id, COUNT(*) as low_stock_count
FROM sync_products WHERE stock_quantity < 10 GROUP BY 1;
```

#### Aggregated Analytics (All Stores)

| Metric | Mô tả |
|--------|-------|
| **Total Revenue** | Tổng doanh thu tất cả cửa hàng |
| **Active Stores** | Số cửa hàng sync trong 7 ngày |
| **Popular Products** | Top SP bán chạy across all stores |
| **Debt Ratio** | % cửa hàng có nợ > X triệu |

### Admin Dashboard Features

1. **Store Overview**: Map các cửa hàng, trạng thái sync
2. **Revenue Charts**: Biểu đồ doanh thu theo vùng miền
3. **Product Trends**: SP trending, seasonal patterns
4. **Alerts**: Cửa hàng inactive, sync failed
