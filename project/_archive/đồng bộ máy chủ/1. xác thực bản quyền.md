# 1. Cơ chế Xác thực Bản quyền & Chống Crack

## Tổng quan
Hệ thống sử dụng mô hình Hybrid Auth:
1.  **Online First**: Ưu tiên xác thực qua Cloud Server để đảm bảo License Key hợp lệ và duy nhất (1 Key = 1 HWID).
2.  **Offline Support**: Sau khi kích hoạt thành công, License Key được lưu cục bộ để hoạt động không cần mạng.

## Quy trình Kích hoạt (Activation Flow)

1.  **Người dùng nhập License Key**.
2.  **App lấy HWID (Hardware ID)**:
    *   Dựa trên Mainboard Serial, CPU ID, MAC Address.
    *   Hash SHA-256 để tạo chuỗi định danh duy nhất (Fingerprint).
3.  **Gửi Request lên Server**:
    *   `POST /api/verify-license`
    *   Body: `{ "license_key": "...", "hwid": "..." }`
4.  **Server Xử lý**:
    *   Kiểm tra Key có tồn tại và còn hạn không.
    *   **Binding**: Nếu Key chưa dùng -> Gán cứng Key này với HWID gửi lên.
    *   **Validate**: Nếu Key đã dùng -> Kiểm tra HWID gửi lên có khớp với HWID đã lưu không.
        *   Khớp: OK (Cho phép cài lại/đổi ổ cứng trên cùng máy).
        *   Không khớp: Hủy (Báo lỗi "Key đã kích hoạt trên thiết bị khác").
5.  **App Nhận Phản hồi**:
    *   Thành công: Lưu file `license.key` vào AppData và mở khóa tính năng Pro.
    *   Thất bại: Báo lỗi cho người dùng.

## 🛡️ Cơ chế Chống Crack & Bảo vệ (Security Layers)

Chúng ta áp dụng chiến lược "Defense in Depth" (Phòng thủ chiều sâu) với 4 lớp bảo vệ:

### 1. Dấu vân tay hệ thống (System Fingerprint - Deep Persistence)
Để ngăn chặn việc gỡ app cài lại ("reset trial"), chúng ta lưu "Ngày cài đặt gốc" (Install Date) vào nhiều nơi sâu trong hệ thống:
*   **Windows Registry**: Lưu tại `HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced\LowLevel\AgriSystemConfig` (Đường dẫn ngụy trang).
*   **Hidden File**: Lưu dự phòng tại `.agri_config` trong thư mục AppData.
*   **Cơ chế**: Khi App khởi động, nó so sánh cả 2 nơi này. Nếu 1 nơi bị xóa, nó sẽ tự khôi phục từ nơi kia. Nếu cả 2 đều mất (cài lại Win), nó mới tính là máy mới.

### 2. Kiểm tra Thời gian (Time Manipulation Check)
Chống lại việc người dùng chỉnh lùi lịch (Backdating) để kéo dài dùng thử:
*   **Logic**: So sánh `Current Time` (giờ hệ thống) với `Last Run Time` (giờ lần cuối mở app) và `Install Date`.
*   **Phát hiện**: Nếu `Current Time` < `Install Date`, hệ thống biết ngay giờ máy đã bị chỉnh lùi -> Khóa App hoặc kích hoạt Chaos Mode.

### 3. Chaos Engine (Động cơ Hỗn loạn)
Thay vì khóa cứng App (dễ bị phát hiện và crack), chúng ta làm App "chập chờn" theo thời gian nếu phát hiện vi phạm (dùng thử quá hạn mà cố bypass):
*   **Ngày 1-7 (Grace Period)**: Hoạt động bình thường.
*   **Ngày 8-30 (Soft Fail)**: Tăng dần xác suất lỗi nhỏ (ví dụ: tạo đơn hàng thỉnh thoảng báo lỗi "System load high", in hóa đơn bị mờ...).
*   **Ngày >30 (Hard Fail)**: Chặn hoàn toàn tính năng Pro, chuyển về Free Mode.
*   **Tác dụng**: Làm nản lòng Cracker và người dùng gian lận mà không cần thông báo "Bạn đang dùng lậu".

### 4. Kill Switch (Công tắc Hủy diệt - Từ xa)
Dành cho trường hợp khẩn cấp (phát hiện Key bị leak công khai):
*   Khi App kết nối mạng để đồng bộ/update, Server có thể gửi lệnh `KILL`.
*   App sẽ nhận lệnh -> Set ngày cài đặt về năm 2020 (hết hạn ngay lập tức) -> Xóa toàn bộ License File -> Buộc người dùng phải liên hệ hỗ trợ.
