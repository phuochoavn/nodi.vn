## 7. 🏗️ Kiến Trúc Auto-Lock Database Cho POS

### 7.1 Database Schema (3NF)

```sql
-- Bảng 1: Hoạt chất (CORE - hạt nhân)
CREATE TABLE tbl_active_ingredients (
    ai_id           TEXT PRIMARY KEY,  -- UUID
    ai_name         TEXT NOT NULL,     -- "Carbosulfan", "Benfuracarb"
    who_toxicity    TEXT CHECK(who_toxicity IN ('Ia','Ib','II','III','U')),
    reg_status      TEXT CHECK(reg_status IN ('Permitted','Transitioning','Banned')),
    circular_ref    TEXT,              -- "75/2025/TT-BNNMT"
    ban_import_date TEXT,              -- ISO 8601
    ban_prod_date   TEXT,              -- ISO 8601
    ban_trade_date  TEXT,              -- ⚡ TRIGGER khóa bán
    penalty_max_vnd INTEGER            -- 50000000
);

-- Bảng 2: Sản phẩm thương phẩm (SKU)
CREATE TABLE tbl_trade_names (
    product_id      TEXT PRIMARY KEY,
    barcode         TEXT UNIQUE,       -- EAN-13
    trade_name      TEXT,              -- "Marshal 200SC"
    manufacturer    TEXT,
    formulation     TEXT CHECK(formulation IN ('EC','SC','WP','GR','WG','EW','OD'))
);

-- Bảng 3: Ánh xạ N-N (1 sản phẩm có nhiều hoạt chất)
CREATE TABLE tbl_product_ai_mapping (
    product_id      TEXT REFERENCES tbl_trade_names(product_id),
    ai_id           TEXT REFERENCES tbl_active_ingredients(ai_id),
    concentration   REAL,  -- mg/kg
    PRIMARY KEY (product_id, ai_id)
);
```

> 🔑 **Quy tắc logic**: 1 sản phẩm map N hoạt chất → chỉ cần **1 hoạt chất Banned** → toàn bộ product bị **Banned**

### 7.3 Cronjob Cập Nhật Động

```
00:00 mỗi ngày:
  1. Quét tbl_active_ingredients
  2. IF CURRENT_DATE >= ban_trade_date
     THEN SET reg_status = 'Banned'
  3. Event Bus → Lock_SKU() cho mọi product_id liên quan
  4. Push notification → Tất cả POS đang kết nối
```
